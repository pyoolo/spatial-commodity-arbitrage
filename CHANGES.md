# Cleanup & Improvements (v0.1.0 → v0.2.0)

This pass focused on three things: **removing committed bloat**, **fixing the
accounting so the headline numbers are honest**, and **general professional
polish**. All 13 tests pass (was 9); the pipeline and figure scripts run clean.

## 1. Repo weight: 4.2 MB → 1.3 MB

The "58% HTML" GitHub reported was **committed generated files**, not
hand-written code. Fixed by not committing artifacts:

- **Removed** `outputs/*.html`, `outputs/*.csv`, `outputs/*.json` (the plotly
  dashboard alone was 752 KB of embedded JS). `outputs/` now holds only a
  `.gitkeep`; everything is regenerated by `make sim`.
- **Stripped notebook outputs**: `notebooks/walkthrough.ipynb` went from 496 KB
  to 12 KB (it was carrying embedded plotly cell output). Kept as source.
- **Added `.gitignore`** covering `outputs/*`, caches, build dirs, notebook
  checkpoints. The README PNGs in `assets/` stay committed so GitHub renders.

The only large files left are the five README PNGs and the compiled paper PDF —
both legitimately part of the published repo.

## 2. Correctness / accounting fixes

### Simulator (`geoarb/simulator.py`) — rewritten
- **Capital locked at booking is now FOB + financing**, not the full delivered
  cost. Freight/handling/tariff settle later and are no longer pre-funded (they
  remain fully in `delivered_cost` and thus in realised PnL). The old behaviour
  overstated working-capital usage and artificially throttled position count.
  New helper `_upfront_capital()`.
- **Equity is marked to market daily.** Open cargoes are revalued at the current
  lane margin instead of frozen at cost, so volatility and drawdown reflect
  in-transit risk rather than discrete settlement jumps. New `unrealised_pnl`
  column on the equity frame.
- **No same-day arrival.** Transit is now `max(1, round(transit_days))`, guarding
  short/zero lanes from settling against their own entry-day price.

**Effect on headline metrics** (seed 42, 750 days, hedged):
Sharpe fell from the artificial **6.0** to a more honest **~3.8**, drawdown
widened from ~5.5% to ~10.5% — as expected once open risk is actually marked.

### Metrics (`geoarb/metrics.py`)
- Renamed `cointegration_check` → **`mean_reversion_check`** (the old name
  over-claimed: it was an OLS AR(1) on a fixed (1,−1) spread, not Engle–Granger).
  `stationary_like` key → `mean_reverting`. Old name kept as a
  backwards-compatible alias.
- **Added `adf_pvalue()`** — a genuine Augmented Dickey–Fuller unit-root test
  (scipy/numpy only, no statsmodels) with a MacKinnon 5% critical value, so
  there's now a real stationarity test behind the claim.

### Equilibrium duals now surfaced (`scripts/run_simulation.py`)
The SPE module's whole selling point is "dual variables = equilibrium location
prices," but the demo never printed them. It now prints supply/demand shadow
prices, plus a **hedged-vs-unhedged** performance comparison that makes the
paper's central claim (the futures hedge removes flat-price risk) visible.
